**SRP - Single Responsibility Prinicple**<br>
In SRP, a class should have only on reason to change. Meaning, a class should do one job only.

**OCP - Open CLose Principle**<br>
Classes, modules, functions etc. should be open for extension but closed for modification. Meaning, one should be able to add a new behavior without modifying the existing code.

**LSP - Liskov Substitution Principle**<br>
Object of a superclass should be replaceable with objects of its subclasses without breaking the program.

**ISP - Interface Segregation Principle**<br>
Subclasses should not be forced to depend on methods they do not use. Meaning, an interface should contain only the methods that a specific client actually needs.

**DIP - Dependency Inversion Principle**<br>
High-level module should not depend directly on concrete implementations. Instead, it should depend on interfaces/abstractions.
